# Render — inchambers-gateway one-click deploy

Render's Blueprint model matches this gateway well: one `render.yaml`
describes the 4 services + Postgres + Redis, and a single Deploy-to-Render
button provisions the whole stack.

## One-click

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/inchambers-ai/inchambers/tree/main/gateway)

After sign-in, Render reads `render.yaml`, shows you the service plan
preview, and asks you to fill the three `sync: false` secrets before
clicking *Apply*:

| Variable | Where to get it |
|---|---|
| `GATEWAY_ORG_ID` (on `ic-gateway-relay`) | inchambers.ai → Org Admin → Settings → *Organization ID* |
| `GATEWAY_MASTER_KEY` | Auto-generated by Render; leave as-is |
| `LITELLM_MASTER_KEY` | Auto-generated by Render; leave as-is |
| `OPENROUTER_API_KEY` (optional) | openrouter.ai/keys |

## After deploy

1. Render assigns `https://ic-gateway-caddy-xxxx.onrender.com` to the public
   service — copy it.
2. Paste it into inchambers.ai → Org Admin → AI Platform → *Gateway URL*.
3. Visit `https://ic-gateway-caddy-xxxx.onrender.com/admin/` and sign in
   with an inchambers.ai org-admin JWT to add ChatGPT Plus / Claude Pro
   cookies via the Providers tab.

## Custom domain

Render → ic-gateway-caddy → Settings → Custom Domains → add
`gateway.yourfirm.com`, set DNS, wait for auto-provisioned TLS.

## Auto-deploy

Render watches `main` by default. Pushes to `inchambers-ai/inchambers`
rebuild the four service images automatically. Pin to a specific commit
or tag under each service's Settings → Build.
